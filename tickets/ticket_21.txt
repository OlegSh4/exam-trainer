Билет №21: Общая структура запроса к базе данных на языке SQL, основные блоки SELECT и последовательность выполнения блоков с группировками данных
## SQL
SQL — это язык запросов к реляционным базам данных.
SQL используется для работы с таблицами: можно получать данные, фильтровать строки, группировать данные, сортировать результат, объединять таблицы, добавлять, изменять и удалять записи.
Самая часто используемая команда SQL — это:
```sql
SELECT

Она нужна для выборки данных из одной или нескольких таблиц.

Простой пример запроса:

SELECT name, age
FROM students;

Этот запрос выбирает столбцы name и age из таблицы students.

Общая структура SELECT-запроса

Общая структура запроса на языке SQL выглядит так:

SELECT столбцы
FROM таблица
WHERE условие
GROUP BY столбцы_группировки
HAVING условие_для_групп
ORDER BY столбцы_сортировки
LIMIT количество_строк;

Не все блоки обязательны.

Обязательными обычно являются только:

SELECT
FROM

Остальные блоки добавляются по необходимости.

Например:

SELECT group_name, COUNT(*) AS student_count
FROM students
WHERE status = 'active'
GROUP BY group_name
HAVING COUNT(*) > 5
ORDER BY student_count DESC;

Этот запрос выбирает активных студентов, группирует их по группе, считает количество студентов в каждой группе, оставляет только группы с количеством студентов больше 5 и сортирует результат по убыванию количества.

Блок SELECT

Блок SELECT указывает, какие столбцы или выражения нужно вывести в результате.

Например:

SELECT name, age
FROM students;

Здесь в результат попадут только столбцы name и age.

Можно выбрать все столбцы с помощью символа *:

SELECT *
FROM students;

Но в реальных задачах лучше явно указывать нужные столбцы, чтобы не получать лишние данные.

Вычисляемые поля в SELECT

В блоке SELECT можно не только выбирать готовые столбцы, но и создавать вычисляемые поля.

Например:

SELECT
    product_name,
    price,
    quantity,
    price * quantity AS total_sum
FROM order_items;

Здесь создается новый столбец total_sum.

Он считается как произведение цены на количество:

price * quantity

Ключевое слово AS задает псевдоним столбца, то есть его новое имя в результате запроса.

Псевдонимы столбцов

Псевдоним нужен, чтобы дать столбцу или выражению понятное имя.

Например:

SELECT
    COUNT(*) AS student_count
FROM students;

Здесь результат подсчета будет называться student_count.

Можно использовать псевдонимы и для обычных столбцов:

SELECT
    name AS student_name,
    group_name AS group_title
FROM students;

Псевдонимы особенно полезны при агрегатных функциях и вычисляемых выражениях.

Блок FROM

Блок FROM указывает, из какой таблицы нужно брать данные.

Например:

SELECT *
FROM students;

Здесь данные берутся из таблицы students.

Если запрос использует несколько таблиц, они также указываются в блоке FROM вместе с соединениями.

Например:

SELECT
    students.name,
    groups.group_name
FROM students
JOIN groups ON students.group_id = groups.group_id;

Здесь данные берутся из таблиц students и groups.

Блок WHERE

Блок WHERE нужен для фильтрации строк.

Он оставляет только те строки, которые удовлетворяют условию.

Например:

SELECT *
FROM students
WHERE age >= 18;

Этот запрос выберет только студентов, возраст которых больше или равен 18.

Еще пример:

SELECT *
FROM students
WHERE group_name = 'PM22-3';

Этот запрос выберет только студентов из группы PM22-3.

Условия в WHERE

В блоке WHERE можно использовать разные операторы сравнения.

Равно:

=

Не равно:

<> 

или в некоторых СУБД:

!=

Больше:

>

Меньше:

<

Больше или равно:

>=

Меньше или равно:

<=

Например:

SELECT *
FROM products
WHERE price > 1000;

Этот запрос выберет товары дороже 1000.

Логические операторы в WHERE

В условиях можно использовать логические операторы.

Оператор AND означает “и”:

SELECT *
FROM students
WHERE age >= 18 AND group_name = 'PM22-3';

Этот запрос выберет студентов старше или равных 18 лет из группы PM22-3.

Оператор OR означает “или”:

SELECT *
FROM students
WHERE group_name = 'PM22-3' OR group_name = 'PM22-4';

Этот запрос выберет студентов из группы PM22-3 или из группы PM22-4.

Оператор NOT означает отрицание:

SELECT *
FROM students
WHERE NOT group_name = 'PM22-3';

Этот запрос выберет студентов не из группы PM22-3.

Оператор BETWEEN

Оператор BETWEEN используется для проверки попадания значения в диапазон.

Например:

SELECT *
FROM products
WHERE price BETWEEN 1000 AND 5000;

Этот запрос выберет товары с ценой от 1000 до 5000 включительно.

То же самое можно записать так:

SELECT *
FROM products
WHERE price >= 1000 AND price <= 5000;

Оператор IN

Оператор IN используется, когда нужно проверить, входит ли значение в список.

Например:

SELECT *
FROM students
WHERE group_name IN ('PM22-3', 'PM22-4', 'PM22-5');

Этот запрос выберет студентов из одной из перечисленных групп.

Без IN пришлось бы писать длинное условие через OR.

Оператор LIKE

Оператор LIKE используется для поиска по шаблону.

Например:

SELECT *
FROM students
WHERE name LIKE 'A%';

Этот запрос выберет студентов, имя которых начинается на букву A.

Символ % означает любое количество любых символов.

Пример:

LIKE '%ov'

означает, что строка заканчивается на ov.

Пример:

LIKE '%ann%'

означает, что строка содержит ann.

Работа с NULL

NULL означает отсутствие значения.

Это не ноль и не пустая строка, а именно неизвестное или отсутствующее значение.

Для проверки NULL нельзя использовать обычное равенство:

column = NULL

Так писать неправильно.

Нужно использовать:

IS NULL

Например:

SELECT *
FROM students
WHERE phone IS NULL;

Этот запрос выберет студентов, у которых не указан телефон.

Для проверки, что значение не пустое:

SELECT *
FROM students
WHERE phone IS NOT NULL;

Блок GROUP BY

Блок GROUP BY используется для группировки строк.

Он нужен, когда мы хотим получить итоговые показатели по группам.

Например, нужно посчитать количество студентов в каждой группе:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
GROUP BY group_name;

Здесь строки группируются по столбцу group_name.

Для каждой группы считается количество строк.

Агрегатные функции

Агрегатные функции считают итоговые значения по группе строк.

Основные агрегатные функции:

COUNT()

считает количество строк.

SUM()

считает сумму.

AVG()

считает среднее значение.

MIN()

находит минимальное значение.

MAX()

находит максимальное значение.

Например:

SELECT
    department,
    AVG(salary) AS avg_salary
FROM employees
GROUP BY department;

Этот запрос считает среднюю зарплату по каждому отделу.

COUNT

Функция COUNT() считает количество строк.

Пример:

SELECT COUNT(*) AS total_students
FROM students;

Этот запрос считает общее количество студентов.

Если нужно посчитать количество непустых значений в конкретном столбце:

SELECT COUNT(phone) AS students_with_phone
FROM students;

COUNT(phone) считает только строки, где phone не равен NULL.

SUM

Функция SUM() считает сумму значений.

Например:

SELECT SUM(amount) AS total_amount
FROM payments;

Этот запрос считает общую сумму платежей.

Можно считать сумму по группам:

SELECT
    client_id,
    SUM(amount) AS client_total
FROM payments
GROUP BY client_id;

Этот запрос считает сумму платежей по каждому клиенту.

AVG

Функция AVG() считает среднее значение.

Например:

SELECT AVG(price) AS avg_price
FROM products;

Этот запрос считает среднюю цену товаров.

Пример по группам:

SELECT
    category,
    AVG(price) AS avg_price
FROM products
GROUP BY category;

Этот запрос считает среднюю цену товаров по каждой категории.

MIN и MAX

Функция MIN() находит минимальное значение.

Функция MAX() находит максимальное значение.

Пример:

SELECT
    MIN(price) AS min_price,
    MAX(price) AS max_price
FROM products;

Этот запрос найдет минимальную и максимальную цену товара.

Пример по группам:

SELECT
    category,
    MIN(price) AS min_price,
    MAX(price) AS max_price
FROM products
GROUP BY category;

Этот запрос найдет минимальную и максимальную цену внутри каждой категории.

Важное правило GROUP BY

Если в запросе есть GROUP BY, то в блоке SELECT можно указывать:

столбцы, по которым идет группировка;

агрегатные функции.

Например, правильно:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
GROUP BY group_name;

Неправильно:

SELECT
    group_name,
    name,
    COUNT(*) AS student_count
FROM students
GROUP BY group_name;

Почему неправильно?

Потому что name не входит в группировку и не является агрегатной функцией.

SQL не понимает, какое именно имя студента нужно вывести для всей группы.

Блок HAVING

Блок HAVING используется для фильтрации групп.

Он похож на WHERE, но работает после группировки.

Например:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
GROUP BY group_name
HAVING COUNT(*) > 5;

Этот запрос сначала группирует студентов по группам, затем считает количество студентов в каждой группе, а потом оставляет только те группы, где студентов больше 5.

Разница между WHERE и HAVING

WHERE фильтрует строки до группировки.

HAVING фильтрует группы после группировки.

Например:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
WHERE status = 'active'
GROUP BY group_name
HAVING COUNT(*) > 5;

Здесь сначала выбираются только активные студенты.

Потом они группируются по группе.

Потом остаются только группы, где активных студентов больше 5.

Простое правило:

WHERE — условие для отдельных строк.

HAVING — условие для групп.

Блок ORDER BY

Блок ORDER BY используется для сортировки результата.

Например:

SELECT *
FROM students
ORDER BY age;

Этот запрос отсортирует студентов по возрасту по возрастанию.

Для сортировки по убыванию используют DESC:

SELECT *
FROM students
ORDER BY age DESC;

Для сортировки по возрастанию можно использовать ASC:

SELECT *
FROM students
ORDER BY age ASC;

ASC часто можно не писать, потому что сортировка по возрастанию используется по умолчанию.

Сортировка по нескольким столбцам

Можно сортировать по нескольким столбцам.

Например:

SELECT *
FROM students
ORDER BY group_name ASC, age DESC;

Этот запрос сначала сортирует студентов по группе, а внутри каждой группы — по возрасту по убыванию.

Это удобно, когда нужно получить аккуратно упорядоченный результат.

Блок LIMIT

Блок LIMIT ограничивает количество строк в результате.

Например:

SELECT *
FROM students
LIMIT 10;

Этот запрос вернет только первые 10 строк.

Часто LIMIT используют вместе с ORDER BY.

Например, чтобы найти 5 самых дорогих товаров:

SELECT *
FROM products
ORDER BY price DESC
LIMIT 5;

Сначала товары сортируются по цене по убыванию, потом берутся первые 5 строк.

DISTINCT

Ключевое слово DISTINCT убирает повторяющиеся значения.

Например:

SELECT DISTINCT group_name
FROM students;

Этот запрос вернет список уникальных групп.

Если написать без DISTINCT:

SELECT group_name
FROM students;

одна и та же группа может повторяться много раз, потому что в ней много студентов.

Последовательность написания SQL-запроса

SQL-запрос обычно пишется в таком порядке:

SELECT
FROM
WHERE
GROUP BY
HAVING
ORDER BY
LIMIT

Пример:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
WHERE status = 'active'
GROUP BY group_name
HAVING COUNT(*) > 5
ORDER BY student_count DESC
LIMIT 10;

Это порядок написания запроса.

Но фактический порядок выполнения блоков другой.

Последовательность выполнения SQL-запроса

Логический порядок выполнения SQL-запроса такой:

FROM
WHERE
GROUP BY
HAVING
SELECT
ORDER BY
LIMIT

Сначала SQL определяет, из какой таблицы брать данные.

Потом фильтрует строки через WHERE.

Потом группирует строки через GROUP BY.

Потом фильтрует группы через HAVING.

Потом вычисляет, какие столбцы вывести через SELECT.

Потом сортирует результат через ORDER BY.

Потом ограничивает количество строк через LIMIT.

Это важно понимать, потому что запрос пишется в одном порядке, а выполняется логически в другом.

Подробный пример выполнения запроса

Рассмотрим запрос:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
WHERE status = 'active'
GROUP BY group_name
HAVING COUNT(*) > 5
ORDER BY student_count DESC;

Сначала выполняется:

FROM students

То есть берется таблица students.

Потом выполняется:

WHERE status = 'active'

Остаются только активные студенты.

Потом выполняется:

GROUP BY group_name

Студенты группируются по группам.

Потом выполняется:

HAVING COUNT(*) > 5

Остаются только группы, где активных студентов больше 5.

Потом выполняется:

SELECT group_name, COUNT(*) AS student_count

Формируется результат: название группы и количество студентов.

Потом выполняется:

ORDER BY student_count DESC

Результат сортируется по убыванию количества студентов.

Почему WHERE выполняется до GROUP BY

WHERE работает со строками исходной таблицы.

Например:

WHERE status = 'active'

Это условие применяется к каждой отдельной строке.

После этого оставшиеся строки попадают в группировку.

То есть группируются уже не все строки таблицы, а только те, которые прошли фильтр WHERE.

Почему HAVING выполняется после GROUP BY

HAVING работает уже не с отдельными строками, а с группами.

Например:

HAVING COUNT(*) > 5

Сначала нужно создать группы и посчитать COUNT(*).

Только потом можно понять, какие группы имеют количество строк больше 5.

Поэтому HAVING выполняется после GROUP BY.

Пример ошибки с WHERE и агрегатной функцией

Нельзя писать агрегатную функцию в WHERE так:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
WHERE COUNT(*) > 5
GROUP BY group_name;

Это ошибка.

Почему?

Потому что WHERE выполняется до группировки, а COUNT(*) появляется только после GROUP BY.

Правильно использовать HAVING:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
GROUP BY group_name
HAVING COUNT(*) > 5;

Группировка по нескольким столбцам

Группировать данные можно сразу по нескольким столбцам.

Например:

SELECT
    group_name,
    status,
    COUNT(*) AS student_count
FROM students
GROUP BY group_name, status;

Этот запрос посчитает количество студентов для каждой пары:

группа;

статус.

Например, отдельно будут посчитаны активные студенты группы PM22-3 и неактивные студенты группы PM22-3.

Агрегация и фильтрация данных

Часто запросы с группировкой используют для аналитики.

Например, нужно посчитать сумму продаж по каждому месяцу:

SELECT
    month,
    SUM(amount) AS total_sales
FROM sales
GROUP BY month
ORDER BY month;

Или найти клиентов, у которых сумма покупок больше 100000:

SELECT
    client_id,
    SUM(amount) AS total_amount
FROM payments
GROUP BY client_id
HAVING SUM(amount) > 100000;

Здесь HAVING нужен, потому что фильтрация идет по результату агрегирования.

Применение SELECT-запросов

SELECT-запросы используются в разных задачах.

В учебной базе данных можно выбирать студентов, группы, оценки и преподавателей.

В интернет-магазине можно анализировать заказы, товары, пользователей и платежи.

В HR-системе можно получать данные по сотрудникам, вакансиям и кандидатам.

В банковской системе можно анализировать клиентов, операции, кредиты и платежи.

В аналитике SELECT-запросы часто используют для построения отчетов и дашбордов.

Краткий ответ для экзамена

SQL — это язык запросов к реляционным базам данных. Основная команда для получения данных — оператор SELECT.

Общая структура SELECT-запроса:

SELECT столбцы
FROM таблица
WHERE условие
GROUP BY столбцы_группировки
HAVING условие_для_групп
ORDER BY столбцы_сортировки
LIMIT количество_строк;

Блок SELECT указывает, какие столбцы или выражения нужно вывести.

Блок FROM указывает, из какой таблицы берутся данные.

Блок WHERE фильтрует отдельные строки до группировки.

Блок GROUP BY группирует строки по указанным столбцам.

Блок HAVING фильтрует уже сформированные группы.

Блок ORDER BY сортирует результат.

Блок LIMIT ограничивает количество строк в результате.

Для группировки данных используют агрегатные функции:

COUNT()
SUM()
AVG()
MIN()
MAX()

Например:

SELECT
    group_name,
    COUNT(*) AS student_count
FROM students
GROUP BY group_name;

Этот запрос считает количество студентов в каждой группе.

Главное отличие WHERE и HAVING состоит в том, что WHERE фильтрует строки до группировки, а HAVING фильтрует группы после группировки.

Порядок написания SQL-запроса:

SELECT
FROM
WHERE
GROUP BY
HAVING
ORDER BY
LIMIT

Логический порядок выполнения запроса:

FROM
WHERE
GROUP BY
HAVING
SELECT
ORDER BY
LIMIT

Это означает, что сначала выбирается источник данных, затем строки фильтруются, потом группируются, затем фильтруются группы, после этого формируется результат, выполняется сортировка и ограничивается количество строк.

Таким образом, оператор SELECT позволяет получать данные из таблиц, фильтровать их, группировать, рассчитывать агрегированные показатели и сортировать результат.
