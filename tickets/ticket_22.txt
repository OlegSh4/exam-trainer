Билет №22: Запросы к базам данных с использованием нескольких таблиц, объединение, пересечение и соединения отношений
## Запросы к нескольким таблицам
В реляционной базе данных информация обычно хранится не в одной большой таблице, а в нескольких связанных таблицах.
Например, в учебной базе могут быть таблицы:
`students` — студенты;
`groups` — группы;
`subjects` — предметы;
`grades` — оценки.
Это делается для того, чтобы не дублировать данные и хранить информацию логично.
Но часто в запросе нужно получить данные сразу из нескольких таблиц.
Например, нужно вывести имя студента и название его группы. Имя хранится в таблице `students`, а название группы — в таблице `groups`.
Для этого в SQL используются операции соединения, объединения и пересечения отношений.
## Отношение в базе данных
В реляционной модели данных таблица называется отношением.
Например, таблица `students` — это отношение, которое хранит данные о студентах.
Строка таблицы называется кортежем или записью.
Столбец таблицы называется атрибутом.
Например:
| student_id | name | group_id |
|---|---|---|
| 1 | Oleg | 10 |
| 2 | Anna | 10 |
| 3 | Ivan | 20 |
Здесь `student_id`, `name`, `group_id` — атрибуты.
Каждая строка — отдельная запись о студенте.
## Связь таблиц
Таблицы связываются между собой с помощью ключей.
Первичный ключ — это поле, которое однозначно определяет строку таблицы.
Например, в таблице `students` первичный ключ:
```sql
student_id

Внешний ключ — это поле, которое ссылается на первичный ключ другой таблицы.

Например, в таблице students поле:

group_id

может ссылаться на поле group_id в таблице groups.

Таблица groups:

group_id	group_name
10	PM22-3
20	PM22-4

Тогда по group_id можно соединить студентов с их группами.

Зачем нужны запросы к нескольким таблицам

Запросы к нескольким таблицам нужны, когда нужная информация распределена по разным таблицам.

Например:

имя студента хранится в таблице students;

название группы хранится в таблице groups;

оценка хранится в таблице grades;

название предмета хранится в таблице subjects.

Чтобы получить полный отчет, нужно соединить эти таблицы.

Например, результат может быть таким:

student_name	group_name	subject_name	grade
Oleg	PM22-3	SQL	5
Anna	PM22-3	Python	4

Такой результат строится через соединения таблиц.

Соединение таблиц

Соединение таблиц в SQL выполняется с помощью оператора JOIN.

Основная идея соединения:

взять строки из одной таблицы;

найти соответствующие строки во второй таблице;

объединить их в одну строку результата.

Пример:

SELECT
    students.name,
    groups.group_name
FROM students
JOIN groups ON students.group_id = groups.group_id;

Этот запрос соединяет таблицы students и groups по полю group_id.

В результате мы получаем имя студента и название его группы.

INNER JOIN

INNER JOIN — это внутреннее соединение.

Оно возвращает только те строки, для которых нашлось совпадение в обеих таблицах.

Пример:

SELECT
    students.name,
    groups.group_name
FROM students
INNER JOIN groups ON students.group_id = groups.group_id;

Если у студента есть group_id, который найден в таблице groups, студент попадет в результат.

Если соответствующей группы нет, такая строка не попадет в результат.

Простыми словами, INNER JOIN оставляет только совпавшие строки.

Пример INNER JOIN

Таблица students:

student_id	name	group_id
1	Oleg	10
2	Anna	10
3	Ivan	30

Таблица groups:

group_id	group_name
10	PM22-3
20	PM22-4

Запрос:

SELECT
    students.name,
    groups.group_name
FROM students
INNER JOIN groups ON students.group_id = groups.group_id;

Результат:

name	group_name
Oleg	PM22-3
Anna	PM22-3

Студент Ivan не попал в результат, потому что в таблице groups нет группы с group_id = 30.

LEFT JOIN

LEFT JOIN — это левое соединение.

Оно возвращает все строки из левой таблицы и подходящие строки из правой таблицы.

Если в правой таблице совпадения нет, то вместо значений правой таблицы будет NULL.

Пример:

SELECT
    students.name,
    groups.group_name
FROM students
LEFT JOIN groups ON students.group_id = groups.group_id;

Здесь левой таблицей является students.

Значит, в результат попадут все студенты, даже если для некоторых не найдется группа.

Пример LEFT JOIN

Таблица students:

student_id	name	group_id
1	Oleg	10
2	Anna	10
3	Ivan	30

Таблица groups:

group_id	group_name
10	PM22-3
20	PM22-4

Запрос:

SELECT
    students.name,
    groups.group_name
FROM students
LEFT JOIN groups ON students.group_id = groups.group_id;

Результат:

name	group_name
Oleg	PM22-3
Anna	PM22-3
Ivan	NULL

Ivan остался в результате, но название группы не найдено, поэтому стоит NULL.

RIGHT JOIN

RIGHT JOIN — это правое соединение.

Оно возвращает все строки из правой таблицы и подходящие строки из левой таблицы.

Если в левой таблице совпадения нет, вместо значений левой таблицы будет NULL.

Пример:

SELECT
    students.name,
    groups.group_name
FROM students
RIGHT JOIN groups ON students.group_id = groups.group_id;

Здесь правой таблицей является groups.

Значит, в результат попадут все группы, даже если в некоторых группах нет студентов.

На практике RIGHT JOIN используют реже, потому что его часто можно заменить на LEFT JOIN, просто поменяв таблицы местами.

FULL JOIN

FULL JOIN — это полное внешнее соединение.

Оно возвращает все строки из обеих таблиц.

Если совпадение найдено, строки соединяются.

Если совпадения нет, отсутствующие значения заполняются NULL.

Пример:

SELECT
    students.name,
    groups.group_name
FROM students
FULL JOIN groups ON students.group_id = groups.group_id;

Такой запрос покажет:

студентов, у которых есть группа;

студентов, у которых группа не найдена;

группы, в которых нет студентов.

FULL JOIN полезен, когда нужно увидеть все данные и найти несовпадения между таблицами.

CROSS JOIN

CROSS JOIN — это декартово произведение таблиц.

Он соединяет каждую строку первой таблицы с каждой строкой второй таблицы.

Если в первой таблице $m$ строк, а во второй $n$ строк, то результат будет содержать:

$$
m\cdot n
$$

строк.

Пример:

SELECT
    students.name,
    subjects.subject_name
FROM students
CROSS JOIN subjects;

Этот запрос создаст все возможные пары “студент — предмет”.

CROSS JOIN нужно использовать осторожно, потому что результат может быть очень большим.

SELF JOIN

SELF JOIN — это соединение таблицы самой с собой.

Оно используется, когда строки одной таблицы нужно сравнить с другими строками этой же таблицы.

Например, есть таблица сотрудников:

employee_id	name	manager_id
1	Ivan	NULL
2	Oleg	1
3	Anna	1

Здесь manager_id указывает на руководителя из той же таблицы.

Чтобы вывести сотрудника и его руководителя, можно использовать SELF JOIN:

SELECT
    e.name AS employee_name,
    m.name AS manager_name
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.employee_id;

Здесь одна и та же таблица используется два раза с разными псевдонимами: e и m.

Псевдонимы таблиц

Псевдонимы таблиц нужны, чтобы писать запрос короче и понятнее.

Например:

SELECT
    s.name,
    g.group_name
FROM students s
JOIN groups g ON s.group_id = g.group_id;

Здесь s — псевдоним таблицы students.

g — псевдоним таблицы groups.

Псевдонимы особенно полезны, когда в запросе несколько таблиц или одна таблица соединяется сама с собой.

Условие соединения ON

Условие ON показывает, по какому правилу соединяются таблицы.

Например:

ON students.group_id = groups.group_id

Это означает, что строка из students соединяется со строкой из groups, если значения group_id совпадают.

Без правильного условия соединения можно получить слишком много строк или неправильный результат.

Типичная ошибка — забыть условие ON. Тогда может получиться декартово произведение таблиц.

Соединение нескольких таблиц

В одном запросе можно соединять больше двух таблиц.

Например, нужно вывести студента, предмет и оценку.

Таблицы:

students;

grades;

subjects.

Запрос:

SELECT
    s.name AS student_name,
    sub.subject_name,
    g.grade
FROM students s
JOIN grades g ON s.student_id = g.student_id
JOIN subjects sub ON g.subject_id = sub.subject_id;

Здесь сначала студенты соединяются с оценками, затем оценки соединяются с предметами.

Так получается полный отчет.

Фильтрация после JOIN

После соединения таблиц можно использовать WHERE.

Например, выбрать только студентов группы PM22-3:

SELECT
    s.name,
    g.group_name
FROM students s
JOIN groups g ON s.group_id = g.group_id
WHERE g.group_name = 'PM22-3';

Сначала таблицы соединяются, потом результат фильтруется по условию.

Агрегация после JOIN

После соединения таблиц можно группировать данные.

Например, посчитать количество студентов в каждой группе:

SELECT
    g.group_name,
    COUNT(s.student_id) AS student_count
FROM groups g
LEFT JOIN students s ON g.group_id = s.group_id
GROUP BY g.group_name;

Здесь используется LEFT JOIN, чтобы в результат попали даже группы без студентов.

Если использовать INNER JOIN, группы без студентов исчезнут из результата.

Объединение отношений

Объединение отношений в SQL выполняется с помощью оператора:

UNION

UNION объединяет результаты двух запросов в один общий результат.

Например, есть две таблицы:

students_2025;

students_2026.

Нужно получить общий список студентов:

SELECT name
FROM students_2025
UNION
SELECT name
FROM students_2026;

Результат будет содержать имена из обеих таблиц.

Особенности UNION

Чтобы использовать UNION, два запроса должны возвращать одинаковое количество столбцов.

Типы соответствующих столбцов должны быть совместимыми.

Например, правильно:

SELECT name, age
FROM students_2025
UNION
SELECT name, age
FROM students_2026;

Неправильно:

SELECT name, age
FROM students_2025
UNION
SELECT name
FROM students_2026;

Здесь в первом запросе два столбца, а во втором один. Такой запрос вызовет ошибку.

UNION и UNION ALL

UNION удаляет дубликаты.

UNION ALL сохраняет все строки, включая повторяющиеся.

Пример:

SELECT name
FROM students_2025
UNION
SELECT name
FROM students_2026;

Если один и тот же студент есть в обеих таблицах, он появится один раз.

А запрос:

SELECT name
FROM students_2025
UNION ALL
SELECT name
FROM students_2026;

оставит все повторения.

UNION ALL обычно работает быстрее, потому что не тратит время на удаление дубликатов.

Пересечение отношений

Пересечение отношений в SQL выполняется с помощью оператора:

INTERSECT

INTERSECT возвращает строки, которые есть в результатах обоих запросов.

Например, нужно найти студентов, которые есть и в списке 2025 года, и в списке 2026 года:

SELECT name
FROM students_2025
INTERSECT
SELECT name
FROM students_2026;

Результат будет содержать только тех студентов, которые встречаются в обеих выборках.

Разность отношений

Хотя в вопросе отдельно указаны объединение и пересечение, рядом часто рассматривают и разность отношений.

Разность показывает строки, которые есть в первом запросе, но отсутствуют во втором.

В SQL для этого часто используют:

EXCEPT

или в некоторых СУБД:

MINUS

Пример:

SELECT name
FROM students_2025
EXCEPT
SELECT name
FROM students_2026;

Этот запрос покажет студентов, которые были в списке 2025 года, но отсутствуют в списке 2026 года.

Отличие JOIN от UNION

JOIN и UNION решают разные задачи.

JOIN соединяет таблицы по столбцам.

То есть он добавляет данные из одной таблицы к данным другой таблицы по условию связи.

Например, к студенту добавляется название его группы.

UNION объединяет строки.

То есть он складывает результаты двух запросов друг под другом.

Например, список студентов из 2025 года объединяется со списком студентов из 2026 года.

Простое правило:

JOIN добавляет столбцы;

UNION добавляет строки.

Отличие JOIN от INTERSECT

JOIN соединяет строки из разных таблиц по условию.

INTERSECT ищет одинаковые строки в результатах двух запросов.

Например, JOIN нужен, чтобы соединить студентов с группами.

INTERSECT нужен, чтобы найти студентов, которые есть в двух списках одновременно.

То есть JOIN связывает данные, а INTERSECT сравнивает два набора строк.

NATURAL JOIN

NATURAL JOIN — это соединение, при котором SQL сам ищет столбцы с одинаковыми именами и соединяет таблицы по ним.

Пример:

SELECT *
FROM students
NATURAL JOIN groups;

Если в обеих таблицах есть столбец group_id, соединение произойдет по нему.

Но на практике NATURAL JOIN используют осторожно.

Причина: если в таблицах появятся другие столбцы с одинаковыми именами, результат может измениться неожиданно.

Обычно безопаснее явно писать условие через ON.

USING

Если столбец для соединения называется одинаково в обеих таблицах, можно использовать USING.

Например:

SELECT
    s.name,
    g.group_name
FROM students s
JOIN groups g USING (group_id);

Это короче, чем:

ON s.group_id = g.group_id

Но ON более гибкий, потому что позволяет задавать любые условия соединения.

Соединение по нескольким условиям

Иногда таблицы нужно соединять сразу по нескольким полям.

Например:

SELECT *
FROM table_a a
JOIN table_b b
ON a.client_id = b.client_id
AND a.date = b.date;

Здесь строки соединяются только если совпадает и client_id, и date.

Такой подход часто используется в аналитических задачах, когда одного поля недостаточно для точного соединения.

Ошибки при соединении таблиц

Первая частая ошибка — забыть условие соединения.

Например:

SELECT *
FROM students, groups;

Такой запрос может дать декартово произведение: каждый студент соединится с каждой группой.

Вторая ошибка — соединить таблицы не по тому полю.

Например, если случайно соединить student_id с group_id, результат будет неверным.

Третья ошибка — использовать INNER JOIN, когда нужно сохранить строки без совпадений.

Например, если нужно показать все группы, включая группы без студентов, нужно использовать LEFT JOIN от таблицы groups.

Четвертая ошибка — не учитывать дубликаты после соединения.

Если в правой таблице несколько подходящих строк, одна строка из левой таблицы может размножиться.

Пример размножения строк при JOIN

Таблица students:

student_id	name
1	Oleg

Таблица grades:

student_id	subject	grade
1	SQL	5
1	Python	4
1	Math	5

Запрос:

SELECT
    s.name,
    g.subject,
    g.grade
FROM students s
JOIN grades g ON s.student_id = g.student_id;

Результат:

name	subject	grade
Oleg	SQL	5
Oleg	Python	4
Oleg	Math	5

Одна строка студента превратилась в три строки, потому что у него три оценки.

Это не ошибка, если такой результат нужен. Но это важно понимать при подсчете количества строк.

COUNT после JOIN

После соединения таблиц нужно аккуратно использовать COUNT.

Например:

SELECT
    s.name,
    COUNT(g.grade) AS grade_count
FROM students s
LEFT JOIN grades g ON s.student_id = g.student_id
GROUP BY s.name;

Этот запрос считает количество оценок у каждого студента.

Если просто написать COUNT(*) после соединения, нужно понимать, что считаются строки уже после JOIN, а не исходные строки таблицы students.

Практический пример с несколькими таблицами

Допустим, нужно получить среднюю оценку каждого студента.

Таблицы:

students(student_id, name);

grades(student_id, subject_id, grade);

subjects(subject_id, subject_name).

Запрос:

SELECT
    s.name,
    AVG(g.grade) AS avg_grade
FROM students s
JOIN grades g ON s.student_id = g.student_id
GROUP BY s.name;

Если нужно вывести еще и предметы:

SELECT
    s.name,
    sub.subject_name,
    g.grade
FROM students s
JOIN grades g ON s.student_id = g.student_id
JOIN subjects sub ON g.subject_id = sub.subject_id;

Краткий ответ для экзамена

В реляционной базе данных данные обычно хранятся в нескольких связанных таблицах. Для получения информации из нескольких таблиц используются соединения, объединения и пересечения отношений.

Соединение таблиц выполняется с помощью оператора JOIN.

Основные виды соединений:

INNER JOIN — возвращает только строки, для которых есть совпадения в обеих таблицах.

LEFT JOIN — возвращает все строки из левой таблицы и подходящие строки из правой таблицы. Если совпадения нет, справа ставится NULL.

RIGHT JOIN — возвращает все строки из правой таблицы и подходящие строки из левой таблицы. Если совпадения нет, слева ставится NULL.

FULL JOIN — возвращает все строки из обеих таблиц, а отсутствующие значения заполняет NULL.

CROSS JOIN — возвращает декартово произведение таблиц, то есть каждую строку первой таблицы соединяет с каждой строкой второй.

SELF JOIN — соединение таблицы самой с собой.

Пример соединения:

SELECT
    s.name,
    g.group_name
FROM students s
JOIN groups g ON s.group_id = g.group_id;

Объединение отношений выполняется оператором UNION. Он объединяет строки результатов двух запросов и удаляет дубликаты:

SELECT name FROM students_2025
UNION
SELECT name FROM students_2026;

UNION ALL также объединяет строки, но сохраняет дубликаты.

Пересечение отношений выполняется оператором INTERSECT. Он возвращает строки, которые есть в обоих результатах:

SELECT name FROM students_2025
INTERSECT
SELECT name FROM students_2026;

Разность отношений часто выполняется через EXCEPT. Она показывает строки, которые есть в первом запросе, но отсутствуют во втором.

Главное отличие JOIN от UNION состоит в том, что JOIN соединяет таблицы по столбцам, добавляя данные из одной таблицы к другой, а UNION объединяет строки двух запросов.

Соединения таблиц применяются для построения отчетов, анализа данных, работы с нормализованными базами данных и получения полной информации, которая хранится в разных таблицах.
